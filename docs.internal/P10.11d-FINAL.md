## Technical Barriers Confirmed

### Barrier 1: Imperative For-Loops (Confirmed by Attempt 2)

The `scan` function uses:
```lean
for _ in [:fuel * 4] do
  match ← scanNextToken s with
  | some s' => s := s'
  | none => return final.tokens  -- Early return!
```

**What we tried**:
```lean
unfold scan at h
simp only [Bind.bind, Pure.pure] at h
-- Result: h contains ForIn operations that don't reduce
```

**Why it failed**:
- ForIn typeclass operations don't have induction principles
- Early return (inside `none` branch) breaks standard loop reasoning
- Mutable state (`s := s'`) creates complex control flow

**What would work**:
- Refactor to explicit structural recursion on fuel parameter
- Then standard induction tactics apply

### Barrier 2: Dependent Array Typing (Confirmed by Attempt 1)

Arrays use dependent types for bounds:
```lean
def Array.getElem (a : Array α) (i : Nat) (h : i < a.size) : α
```

After `push`, indices change type:
- Before: `Fin s.tokens.size`
- After: `Fin (s.tokens.size + 1)`

**What we tried**:
```lean
by_cases hj : j.val < s.tokens.size
case pos =>
  have hi : i.val < s.tokens.size := ...
  exact h_ordered ⟨i.val, hi⟩ ⟨j.val, hj⟩ hij  -- Type error!
```

**Why it failed**:
Even though both refer to the same array element semantically, Lean cannot coerce between `Fin n` and `Fin (n+1)` automatically in this context.

**What would work**:
- Restate theorem using `Nat` indices with separate bounds
- Or develop extensive Fin coercion lemma library

## Options Re-Evaluated

### Option A: Refactor to Structural Recursion ✅ VIABLE

Replace imperative loop with explicit recursion:

```lean
def scanHelper (s : ScannerState) (fuel : Nat) :
    Except ScanError (Array (Positioned YamlToken)) :=
  match fuel with
  | 0 => .error (.fuelExhausted ...)
  | fuel' + 1 =>
    match scanNextToken s with
    | some s' => scanHelper s' fuel'
    | none =>
      let final := (unwindIndents s (-1)).emit .streamEnd
      .ok final.tokens
```

**Proof becomes**:
```lean
theorem scan_produces_at_least_two ... := by
  -- Induction on fuel
  induction fuel with
  | zero => contradiction  -- .error, not .ok
  | succ fuel' ih =>
    -- Case on scanNextToken result
    cases scanNextToken s with
    | some s' => apply ih  -- Inductive case
    | none =>
      -- Final path: use mk'_tokens_empty + emit_tokens_size
      calc
        tokens.size = (final.emit .streamEnd).tokens.size := ...
        _           = final.tokens.size + 1 := emit_tokens_size
        _           ≥ 1 + 1 := ...  -- final has streamStart
        _           = 2 := rfl
```

**Effort**: 7-12 days (2-3 refactor + 5-9 proving)
**Risk**: Medium (code changes to production scanner)
**Confidence**: High (standard induction techniques work)
