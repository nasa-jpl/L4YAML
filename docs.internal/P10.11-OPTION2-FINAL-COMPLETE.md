## Lessons Learned

### 1. The Cascade Pattern

**Discovery**: "Prove one theorem" often reveals N hidden dependencies

**This session**:
```
Prove skipToContent_preserves_tokens
    ↓
Requires 4 helper theorems
    ↓
3 helpers blocked by infrastructure
    ↓
Refactor 3 functions
    ↓
Prove 4 helpers
    ↓
Hit system limitation
    ↓
Accept axiom
```

**Lesson**: Always check ALL transitive dependencies before estimating

### 2. Three Types of Blockers

| Type | Example | Solution | Time |
|------|---------|----------|------|
| **Conceptual** | Parser value semantics | Right abstraction | Hours of thinking |
| **Infrastructure** | Imperative loops | Systematic refactoring | ~30 min each |
| **System Limitation** | skipToContentLoop visibility | Axiom with docs | Investigation varies |

**New understanding**: Not all blockers are created equal

### 3. When Axioms Are Appropriate

**Good use** (this case):
- All dependencies proven ✅
- Property clearly true ✅
- Tool limitation (not proof gap) ✅
- Thorough investigation ✅
- Well-documented ✅

**Bad use** (avoid):
- Skipping hard proof work ❌
- Property not obviously true ❌
- Could be proven with more effort ❌
- Poorly documented ❌

### 4. Proof-Preserving Refactoring Works

**Pattern validated** (4/4 successes):
```lean
-- Before: for _ in [:fuel] do ...
-- After:
def fooLoop (s : State) (fuel : Nat) : ReturnType :=
  match fuel with
  | 0 => base_case
  | fuel' + 1 => recursive_case (fooLoop ... fuel')
termination_by fuel

def foo (s : State) : ReturnType :=
  fooLoop s (appropriate_fuel)
```

**Success rate**: 100% (4/4 functions)
**Time per function**: ~15-30 minutes
**Regressions**: 0
