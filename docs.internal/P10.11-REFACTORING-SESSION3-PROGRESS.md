## Detailed Refactorings

### 1. scanAnchorOrAlias ✅

**Challenge**: Imperative for-loop collecting name characters until flow indicator/whitespace/linebreak.

**Solution**: Structural recursion helper `collectAnchorNameLoop` with fuel-based termination.

**Pattern**:
```lean
-- Helper: Structural recursion with fuel
private def collectAnchorNameLoop (s : ScannerState) (name : String) (fuel : Nat) : String × ScannerState :=
  match fuel with
  | 0 => (name, s)
  | fuel' + 1 =>
    match s.peek? with
    | some c =>
      if !isFlowIndicator c && !isWhiteSpace c && !isLineBreak c then
        collectAnchorNameLoop s.advance (name.push c) fuel'
      else
        (name, s)
    | none => (name, s)

-- Main function: Explicit variable names
def scanAnchorOrAlias (s : ScannerState) (isAnchor : Bool) : ScannerState :=
  let startPos := s.currentPos
  let s_after_marker := s.advance
  let fuel := s.inputEnd - s_after_marker.offset
  let (name, s_after_name) := collectAnchorNameLoop s_after_marker "" fuel
  let token := if isAnchor then YamlToken.anchor name else YamlToken.alias name
  let s_with_token := s_after_name.emitAt startPos token
  { s_with_token with simpleKeyAllowed := false }
```

### 2. scanTag ✅

**Challenge**: Complex control flow with 3 branches, each having imperative loops:
1. Verbatim tag: `!<uri>` - loop to collect URI
2. Secondary tag: `!!suffix` - loop to collect suffix
3. Named/primary tag: `!handle!suffix` or `!suffix` - multiple loops

**Solution**: Three structural recursion helpers:

```lean
-- Helper 1: Collect verbatim URI until '>'
private def collectVerbatimTagLoop (s : ScannerState) (uri : String) (fuel : Nat) : String × ScannerState :=
  match fuel with
  | 0 => (uri, s)
  | fuel' + 1 =>
    match s.peek? with
    | some '>' => (uri, s.advance)
    | some c => collectVerbatimTagLoop s.advance (uri.push c) fuel'
    | none => (uri, s)

-- Helper 2: Collect tag suffix (non-whitespace, non-flow)
private def collectTagSuffixLoop (s : ScannerState) (suffix : String) (fuel : Nat) : String × ScannerState :=
  match fuel with
  | 0 => (suffix, s)
  | fuel' + 1 =>
    match s.peek? with
    | some c =>
      if !isWhiteSpace c && !isLineBreak c && !isFlowIndicator c then
        collectTagSuffixLoop s.advance (suffix.push c) fuel'
      else
        (suffix, s)
    | none => (suffix, s)

-- Helper 3: Collect handle until '!' or invalid char
private def collectTagHandleLoop (s : ScannerState) (chars : String) (fuel : Nat) : String × Bool × ScannerState :=
  match fuel with
  | 0 => (chars, false, s)
  | fuel' + 1 =>
    match s.peek? with
    | some '!' => (chars, true, s.advance)
    | some c =>
      if !isWhiteSpace c && !isLineBreak c && !isFlowIndicator c then
        collectTagHandleLoop s.advance (chars.push c) fuel'
      else
        (chars, false, s)
    | none => (chars, false, s)
```

**Main function** refactored with explicit variable names for each branch.

## Key Technical Achievements

### Achievement 1: Structural Recursion Pattern Established

**Pattern for all imperative for-loops**:
```lean
private def collectXyzLoop (s : ScannerState) (acc : String) (fuel : Nat) : Result :=
  match fuel with
  | 0 => (acc, s)  -- Fuel exhausted
  | fuel' + 1 =>
    match s.peek? with
    | some c =>
      if condition then
        collectXyzLoop s.advance (acc.push c) fuel'  -- Continue
      else
        (acc, s)  -- Break on condition
    | none => (acc, s)  -- EOF
```

**Key insight**: `match fuel with | 0 => ... | fuel' + 1 => ...` provides automatic termination proof.

**scanTag** demonstrates handling:

### Insight 1: Complex Functions Follow Same Pattern

**Finding**: Even the most complex function (scanDirective) follows the same refactoring pattern.

**Evidence**:
- scanDirective has 5 loops → 5 structural recursion helpers
- Each helper: ~12 lines (match fuel, handle conditions, recurse)
- Main function: explicit variable names throughout

### Insight 2: Structural Recursion Is Mechanical

**Finding**: Converting imperative for-loops to structural recursion is now a mechanical process.

**Pattern**:
1. Identify loop accumulator and termination
2. Create helper with `fuel` parameter
3. Pattern match on `fuel` (0 or fuel'+1)
4. Handle peek conditions (some c, none)
5. Recurse with `fuel'` or return

**Time**: ~15-30 minutes per helper function

### Insight 3: Reserved Words Are Subtle

**Finding**: Parameter names matter - `prefix` is reserved.

**Solution**: Use domain-appropriate alternatives (`pfx`, `tagPrefix`, etc.)
