### Hypotheses for the Failure

1. **Return Type Difference**: `skipToContentLoop` returns `Except ScanError ScannerState` (monadic), while the other loops return plain `ScannerState`. This may affect how Lean resolves names.

2. **Definition Order**: `skipToContentLoop` is defined later in Scanner.lean (line 416) compared to the other loops (lines 350-401). Module compilation order might matter.

3. **Visibility Rules**: There may be different visibility rules for definitions that return monadic types vs. plain types.

4. **Build System Issue**: Possible caching or compilation artifact issue (though `lake clean` and rebuild doesn't fix it).

## Lessons Learned

### 1. Visibility Is Not Obvious

**Observation**: Four functions defined with near-identical patterns in the same module, but one is not accessible in proofs while the others are.

**Learning**: Lean's visibility and name resolution rules for definitions in the same module are subtle. The return type (`Except` vs. plain type) may affect visibility.

### 2. Infrastructure Refactoring Pattern Validated

**Success**: The refactoring pattern (imperative loop → structural recursion) worked perfectly for all 4 functions and enabled all 4 helper proofs.

**Pattern**:
```lean
-- Before: for _ in [:fuel] do ...
-- After:
def fooLoop (s : State) (fuel : Nat) : ReturnType :=
  match fuel with
  | 0 => base_case
  | fuel' + 1 => recursive_case (fooLoop ... fuel')
termination_by fuel

def foo (s : State) : ReturnType :=
  fooLoop s (s.inputEnd - s.offset)
```

### 3. Proof Pattern Consistency

**Success**: The proof pattern (unfold, generalize fuel, induct, split/rw) worked identically for 3/4 functions.

**Pattern**:
```lean
theorem foo_preserves_tokens :
  unfold foo
  generalize h_fuel : ... = fuel
  induction fuel generalizing s with
  | zero => unfold fooLoop; rfl
  | succ fuel' IH =>
    unfold fooLoop
    split
    · -- use IH and helper lemmas
    · rfl
```

### 4. Monadic Returns May Complicate Proofs

**Hypothesis**: Functions returning `Except` may have different proof requirements than those returning plain values. The theorem signature changes from:
```lean
-- Plain: (foo s).tokens = s.tokens
-- Except: foo s = .ok s' → s'.tokens = s.tokens
```

This adds an extra layer of indirection that may interact poorly with the visibility system.
