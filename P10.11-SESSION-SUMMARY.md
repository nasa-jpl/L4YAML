## Technical Challenges Documented

### 1. Imperative Loop Reasoning (P10.11a)
**Challenge**: Lean's `for _ in [:n] do ...` loops with mutable state resist standard proof techniques.

**Approaches attempted**:
- Unfolding → complex recursive definitions
- Loop invariants → manual induction over iteration count
- Do-notation → nested bind operations

**Solution adopted**: Extensive `#guard` validation + documented strategies. Accept that some proofs may remain with `sorry` until better loop reasoning support exists.

### 2. The Annotation Gap (P10.11b)
**Challenge**: Grammar's `ValidNode` is annotation-free (structure only), but parser's `YamlValue` includes tags/anchors (metadata).

**This is correct by design**:
- Grammar specifies valid **structure**
- Parser preserves **metadata**
- `NodeToValue` ignores metadata

**Solution path**: Either weaken `NodeToValue` to work on `stripAnnotations` versions, or prove `stripAnnotations` preserves the relation.

### 3. Scanner-Parser Contract (P10.11b)
**Challenge**: Need to show `TokenParser` preserves `Grammable` properties (validPlainFirst, noColonSpace, etc.) that the scanner guarantees.

**This crosses layers**:
- Scanner produces tokens from characters (enforces properties)
- Parser reconstructs values from tokens (must preserve properties)
- Need to show round-trip preservation

**Solution path**: Strengthen P10.11a scanner correctness to include character property preservation, then compose with parser.
