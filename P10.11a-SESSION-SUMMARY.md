## Proof Patterns Discovered

### Pattern 4: 3-Case Split for peek? Matches

```lean
theorem loop_preserves_tokens (s : ScannerState) (acc : α) (fuel : Nat) :
    (loop s acc fuel).snd.tokens = s.tokens := by
  induction fuel generalizing s acc with
  | succ fuel' ih =>
    unfold loop
    split
    · -- some specific_char (e.g., '>' for verbatim tag)
      simp only []; exact advance_preserves_tokens s
    · -- some c (generic character)
      rw [ih]; exact advance_preserves_tokens s
    · -- none
      simp only []
```

**Used in:** collectVerbatimTagLoop, collectTagSuffixLoop, collectTagHandleLoop

**Key insight:** Always handle three cases, not two. The match splits into `some <specific>`, `some <other>`, and `none`.

### Pattern 5: Escape Sequence Proof Chain

```lean
-- 1. Base: hex digit collection preserves tokens
theorem collectHexDigitsLoop_preserves_tokens : ...

-- 2. Parse uses collectHexDigits
theorem parseHexEscape_preserves_tokens : ... := by
  unfold parseHexEscape
  have h_collect := collectHexDigitsLoop_preserves_tokens ...
  ...

-- 3. Process uses parseHexEscape
theorem processEscape_preserves_tokens : ... := by
  unfold processEscape
  have h_parse := parseHexEscape_preserves_tokens ...
  repeat (split at h)
  all_goals (first | ... | ...)

-- 4. Main loop uses processEscape
theorem collectDoubleQuotedLoop_preserves_tokens : ... := by
  have h_proc := processEscape_preserves_tokens ...
  ...
```

**Key insight:** Build proof chains from leaves to root, each layer depending on the previous.

### Pattern 6: repeat + all_goals for Many Similar Cases

```lean
theorem processEscape_preserves_tokens (s : ScannerState) (ch : Char) (s' : ScannerState)
    (h : processEscape s = .ok (ch, s')) :
    s'.tokens = s.tokens := by
  unfold processEscape at h
  simp only [] at h
  split at h <;> try contradiction
  repeat (split at h)
  all_goals (
    first
    | (injection h with h_eq; cases h_eq; exact advance_preserves_tokens s)
    | (have h_adv := advance_preserves_tokens s
       have h_hex := parseHexEscape_preserves_tokens s.advance _ ch s' h
       rw [h_hex, h_adv])
    | contradiction
  )
```

**Used in:** processEscape_preserves_tokens (handles 12+ escape character cases)

**Key insight:** When a function has many similar branches (escape characters, validation checks), use `repeat (split)` then `all_goals` with `first` to apply the right pattern to each case.

## Key Insights

### What Worked Exceptionally Well

1. **Proof chain strategy** — Building escape sequence proofs from leaves (collectHexDigits) to root (collectDoubleQuoted) worked perfectly

2. **Pattern reuse** — Once collectPlainScalarLoop was proven, the other two collect loops followed the same structure with adjustments

3. **repeat + all_goals pattern** — Discovered this for processEscape; handles 12+ similar cases elegantly

4. **3-case split discipline** — Always splitting peek? matches into `some specific`, `some c`, `none` prevented subtle bugs

### What Was Challenging

1. **collectPlainScalarLoop complexity** — Most intricate proof due to:
   - Flow vs block context handling
   - skipBlankLinesLoop with nested conditionals
   - Document boundary checks
   - **Lesson:** Complex state machines need explicit case-by-case split handling

2. **Monadic validation branches** — Functions with multiple validation checks (atDocumentStart, underIndented, etc.) require careful split sequencing
   - **Solution:** Split on each validation, use `try contradiction` to eliminate error paths

3. **scanDocumentEnd complexity** — Deferred due to nested pure () branches making split difficult
   - **Lesson:** Not all lemmas need to be proven immediately; focus on critical path

### Process Improvements

1. ✅ **Deferred non-critical proofs** — scanDocumentEnd and scanDirective marked as sorry to focus on critical path

2. ✅ **Infrastructure-first approach** — Proving all collect loops before scan* functions paid off

3. ✅ **Documentation of patterns** — 6 patterns now documented (was 3)

### Technical Discoveries

**Split Strategy for peek? matches:**
