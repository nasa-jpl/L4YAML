## Main Results

```lean
-- Top-level specification
def ValidYaml (input : String) (docs : Array YamlDocument) : Prop :=
  ∃ (vts : ValidTokenStream),
    vts.input = input ∧
    Scanner.scan input = .ok vts.tokens ∧
    TokenParser.parseStream vts.tokens = .ok docs ∧
    ∀ doc ∈ docs.toList, ∃ node : ValidNode, NodeToValue node doc.value

-- Soundness: successful parsing implies grammar validity
theorem parse_sound :
  TokenParser.parseYaml input = .ok docs → ValidYaml input docs

-- Completeness: grammar validity implies parsing succeeds
theorem parse_complete :
  ValidYaml input docs → TokenParser.parseYaml input = .ok docs
```

**Meaning**: These theorems make the aspirational examples from Grammar.lean:533-538 into real declarations. They establish that the parser implementation respects the grammar specification.

## Architecture

The end-to-end correctness composes three layers:

```
String --[scan]--> ValidTokenStream --[parseStream]--> Array YamlDocument --[witnesses]--> ValidNode
   ^                      ^                                    ^                            ^
   |                      |                                    |                            |
 input              P10.11a infrastructure              P10.11b infrastructure         Grammar spec
```

### ValidYaml Definition

`ValidYaml` bridges the three layers by asserting:
1. The input string can be scanned to tokens (scanner layer)
2. The tokens form a `ValidTokenStream` (scanner correctness, P10.11a)
3. The tokens can be parsed to documents (parser layer)
4. Each document has a `ValidNode` witness (parser correctness, P10.11b)

This definition is compositional — it's defined entirely in terms of the results from P10.11a and P10.11b.

### 2. `parse_complete` (with `sorry`)
**Claim**: Grammar validity implies parsing succeeds

```lean
theorem parse_complete (input : String) (docs : Array YamlDocument)
    (h : ValidYaml input docs) : TokenParser.parseYaml input = .ok docs
```

**What's needed**: Given `ValidYaml`, extract:
- `vts : ValidTokenStream` with `vts.input = input`
- `h_scan : scan input = .ok vts.tokens`
- `h_parse : parseStream vts.tokens = .ok docs`

Then show `parseYaml input = .ok docs` by rewriting with these components.

**Challenge**: `parseYaml` composes documents via `.map YamlDocument.compose`, so we need to show either:
1. The `docs` in `ValidYaml` are already composed, or
2. The composition step doesn't affect the result

This is the harder direction because we're constructing the parse result from the specification.

### 4. `parse_deterministic` (proven)
**Claim**: Parsing is deterministic

```lean
theorem parse_deterministic :
  TokenParser.parseYaml input = .ok docs₁ →
  TokenParser.parseYaml input = .ok docs₂ →
  docs₁ = docs₂
```

**Status**: ✅ **Complete proof** (no `sorry`)
- Pure function, so two successful parses must be equal
- Proven by transitivity and `Except.ok` injection
