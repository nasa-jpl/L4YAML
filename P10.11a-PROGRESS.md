## Pattern Library

### Pattern 1: Structural Recursion on Fuel

```lean
theorem loop_preserves_tokens (s : ScannerState) (acc : α) (fuel : Nat) :
    (loop s acc fuel).snd.tokens = s.tokens := by
  induction fuel generalizing s acc with
  | zero => unfold loop; rfl
  | succ fuel' ih =>
    unfold loop
    split
    · split
      · rw [ih]; exact advance_preserves_tokens s
      · rfl
    · rfl
```

**Used in:** `collectAnchorNameLoop_preserves_tokens`

### Pattern 2: Monadic Do-Notation

```lean
theorem scan*_adds_token (s : ScannerState) (s' : ScannerState)
    (h : scan* s = .ok s') :
    s'.tokens.size = s.tokens.size + 1 := by
  unfold scan* at h
  simp only [bind, Except.bind] at h  -- Expose Except.bind matches
  split at h <;> try contradiction    -- Split on .error/.ok branches

  rw [emitAt_tokens_size]             -- Apply size lemma
```

**Pattern:** `simp only [bind, Except.bind]` → `split` → `contradiction` → `injection`

### Pattern 3: Structure Update Preservation

```lean
-- When function returns { s_modified with field := value }

rw [h]
```

**Used in:** token size proofs where only non-token fields are updated

## Lessons Learned

### Unexpected Challenges

1. **Do-notation opacity**: `do` blocks desugar to `Bind.bind`, not `Except.bind`.
   - **Solution**: `simp only [bind, Except.bind]` exposes the match structure
   - Without this, `split at h` cannot find the conditional branches

2. **Structural recursion with advance**: Proving loop functions preserve tokens requires:
   - Induction on fuel parameter
   - Applying `advance_preserves_tokens` at recursive call sites
   - Pattern: `rw [ih]; exact advance_preserves_tokens s`

3. **Structure updates**: When a function returns `{ s with field := value }`, the projection
   `result.tokens` doesn't automatically simplify to `s.tokens`.
   - **Solution**: `have h : result.tokens = s.tokens := by cases h_result; rfl`

### Simplifications

1. **Private helper lemmas**: Using `private theorem` keeps the proof namespace clean
   while still allowing composition within the file.

2. **Existing infrastructure reuse**: Lemmas like `emit_tokens_size`, `emitAt_tokens_size`,
   `advance_preserves_tokens` are already proven. Most new proofs are just:
   - Unfold the function
   - Expose the monad structure
   - Apply existing lemmas
   - Omega for arithmetic

3. **Pattern-driven development**: Once `collectAnchorNameLoop_preserves_tokens` was proven,
   the pattern for other loop functions was clear.

### Idioms

1. **`simp only [bind, Except.bind] at h` before `split at h`**
   - Essential for proving properties of monadic functions
   - Transforms opaque do-blocks into matchable conditionals

2. **`injection h with h_eq; subst h_eq`**
   - Standard pattern for extracting values from `Except.ok` constructor
   - Cleaner than match-based extraction

3. **`split at h <;> try contradiction`**
   - Handles all error branches at once
   - Only leaves the success path open

4. **Helper lemma naming**: `function_operation_property`
   - Examples: `scanKey_adds_tokens`, `collectLoop_preserves_tokens`
   - Makes proof structure self-documenting
