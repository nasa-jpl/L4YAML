### 3. `parseStream_respects_grammar` (with `sorry`)
**Claim**: Main result connecting parser to grammar

```lean
theorem parseStream_respects_grammar :
  parseStream tokens = .ok docs →
  ValidTokenStream input tokens →
  ∀ doc ∈ docs, ∃ node, NodeToValue node doc.value
```

**What's needed**: Bridge the gap between:
- **What we have**: `stripAnnotations (toYamlValue node) = stripAnnotations doc.value`
- **What we need**: `NodeToValue node doc.value`

**Challenge**: The `stripAnnotations` relation is weaker than direct equality. The grammar's `NodeToValue` doesn't handle annotations (tags/anchors), so we need to either:
1. Define a weaker `NodeToValue` that ignores annotations, or
2. Prove `stripAnnotations` preserves the `NodeToValue` relation

## Technical Insight: The Annotation Gap

The key challenge is that:
- `ValidNode` in Grammar.lean doesn't model tags/anchors (it's pure structure)
- `YamlValue` in Types.lean includes tags/anchors (it's the runtime representation)
- `NodeToValue` connects structure to values **without annotations**
- Parser output includes annotations from the input

This is actually **correct by design**:
- The grammar specifies **structure** (what parses are valid)
- Tags/anchors are **metadata** (not part of the grammar)
- The parser preserves metadata while respecting structural grammar

**Solution path**: Define `NodeToValue` to work on `stripAnnotations` versions:
```lean
theorem nodeToValue_ignores_annotations :
  stripAnnotations v₁ = stripAnnotations v₂ →
  NodeToValue n v₁ →
  NodeToValue n v₂
```

Or alternatively, weaken the main theorem to only claim correspondence after stripping.
