    Expression Search Utilities #

    Shared utilities for searching and extracting expressions from Lean's InfoTree. Used by DataFlow, EffectFlow, and SemanticTableau display styles.

              Outermost Expression Finding #

              source
              def LeanPrism.SearchExpression.findOutermostTerm (terms : Array CollectedTerm) (text : Lean.FileMap) (cursor : Lean.Lsp.Position) (ruleApplies : Lean.Expr → Bool) :
              Except GraphError CollectedTerm

              Find the outermost expression matching a predicate, containing the cursor position. Uses expression containment to determine hierarchy when position info is incomplete. For monadic expressions, source positions may only cover the immediate syntax, not continuations. We include all terms that start at or before the cursor line. Additionally, if the found term is inside a match expression, prefer the match.

                Current Node Selection #

                def LeanPrism.SearchExpression.findCurrentNodeId (nodes : Array GraphNode) (text : Lean.FileMap) (cursor : Lean.Lsp.Position) :
                Option Nat

                Find the current node ID based on cursor position using range containment. Uses the pattern from Lean 4's hoverableInfoAtM?:

                1. Find all nodes whose range contains the cursor
                2. Prefer the smallest (most specific) node Skips nodes at position (0,0) which indicates no real position was assigned.
