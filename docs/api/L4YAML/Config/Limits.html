    structure L4YAML.AliasLimits :
    Type

    Limits on alias (anchor/*ref) resolution. Prevents billion-laugh exponential expansion and cyclic references.

    • maxAliasDepth : Nat

      Maximum depth of alias resolution chains. Example: a: &a *b, b: &b *c, c: "x" → depth 3. Default: 50

    • maxAliasExpansions : Nat

      Maximum total alias substitution steps per document. Each .alias → value replacement counts as one. Default: 10,000

    • maxResolvedNodes : Nat

      Maximum total nodes in the resolved tree. Default: 100,000

    • rejectCycles : Bool

      Detect and reject cyclic aliases (a: &a [*a]). Default: true

            Type

              Maximum nesting depth of collections. Default: 100

            • maxSequenceLength : Nat

            • maxMappingSize : Nat

            • maxScalarBytes : Nat

            • maxTotalNodes : Nat

                            inductive L4YAML.TagPolicy :
                            Type

                            Tag validation policy.

                            • allowAll : TagPolicy

                              Accept all tags (UNSAFE — only for trusted input).

                            • rejectAll : TagPolicy

                              Reject all explicit tags; only implicit typing allowed.

                            • whitelist (allowed : List String) : TagPolicy

                              Only accept tags in the allowed list.

                            • blacklist (forbidden : List String) : TagPolicy

                              Reject tags whose prefix matches the forbidden list.

                            • coreSchemaOnly : TagPolicy

                              Only YAML 1.2 Core Schema tags.
