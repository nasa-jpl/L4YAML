    Scalar Surface Syntax — Chapters 7.3 & 8.1 #

    Surface syntax predicates for YAML scalar styles: double-quoted, single-quoted, plain, literal block, and folded block scalars.

    Productions Covered #

    • Double-quoted: [107]-[114] c-double-quoted, nb-double-text, nb-double-char, nb-double-one-line, s-double-escaped, s-double-break, nb-double-multi-line
    • Single-quoted: [118]-[122] c-single-quoted, nb-single-text, nb-single-char, nb-single-one-line, nb-single-multi-line
    • Plain scalars: [123]-[133] ns-plain, ns-plain-first, ns-plain-safe, ns-plain-char, ns-plain-one-line, s-ns-plain-next-line, ns-plain-multi-line
    • Block scalars: [162]-[179] c-l+literal, c-l+folded, l-nb-literal-text, b-nb-literal-next, l-literal-content, s-nb-folded-text, etc.

                                              §5 Block Scalars [162]–[179] #

                                              Block scalars are the most complex scalar form: a header line (with chomp and indent indicators) followed by content lines at a detected indentation level. The indentation auto-detection makes the grammar context-dependent in a way that's naturally expressed as existential quantification over the indent value.
