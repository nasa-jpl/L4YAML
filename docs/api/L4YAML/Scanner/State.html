    Scanner State #

    The ScannerState structure and its low-level accessors — the substrate on which every other scanner submodule operates.

    Split from the monolithic Scanner.lean during Blueprint Initiative 1 Phase 2 (Scanner split). See Blueprint/03-code-organization.md.

    Scope #

    • Structural types: IndentEntry, SimpleKeyState, ScannerState.
    • Structural well-formedness invariant: ScannerState.WellFormed.
    • Initial-state constructor: ScannerState.mk'.
    • Navigation accessors: peek?, peekAt?, peekBack?, advance, advanceN, hasMore, currentPos, inFlow, isInFlowSequence, currentIndent.
    • Token emission: emit, emitAt.

    Nothing in this file consumes characters in a spec-productions sense — those live in Scanner/Whitespace.lean, Scanner/Indent.lean, etc.

                    structure L4YAML.Scanner.ScannerState :
                    Type

                    The scanner's mutable state.

                    • input : String

                      Source string being scanned

                    • inputEnd : Nat

                      End byte offset of input (cached from input.utf8ByteSize)

                    • offset : Nat

                      Current byte offset

                    • line : Nat

                      Current line (0-based)

                    • col : Nat

                      Current column (0-based)

                    • indents : Array IndentEntry

                      Indentation stack. Bottom entry: { column := -1 }.

                    • flowLevel : Nat

                      Flow nesting level. 0 = block context.

                    • tokens : Array (Positioned YamlToken)

                      Emitted tokens

                    • simpleKey : SimpleKeyState

                      Simple key tracking

                    • simpleKeyStack : Array SimpleKeyState

                      Stack of saved simple keys for enclosing flow nesting levels. Pushed on flow-open ([, {), popped on flow-close (], }). This allows a simple key (e.g., a flow mapping used as a key: {a: b}: val) to survive nested flow collection scanning.

                    • simpleKeyAllowed : Bool

                      Whether a simple key is allowed at the current position. Set true after line breaks, block entries, keys, values. Set false after scalars, anchors, aliases, tags.

                    • needIndentCheck : Bool

                      Whether we need to check indentation at current position

                    • allowDirectives : Bool

                      Whether directives (%YAML, %TAG) are allowed at the current position. True at stream start and after ...; false after --- or content.

                    • seenYamlDirective : Bool

                      Whether a %YAML directive has been seen in the current directive block. Reset when --- is emitted. Used to detect duplicate %YAML.

                    • directivesPresent : Bool

                      Whether any directive has been emitted since last ---. Used to detect directives-without-document at EOF and before ....

                    • documentEverStarted : Bool

                      Whether a document has ever been started (via --- or implicit content). Used to detect directive-only streams.

                    • flowStack : Array Bool

                      Stack tracking flow collection types. true = sequence ([), false = mapping ({). Pushed on flow-open, popped on flow-close.

                    • explicitKeyLine : Option Nat

                      Line number of the most recent explicit ? key indicator, if the entry has not yet been closed by :. Used to (a) inhibit simple-key saving on the same line as ?, and (b) bypass the flow-sequence implicit-key-single-line restriction when ? was used.

                    • comments : Array (YamlPos × String)

                      Collected comments (position × text). Comments are stored here as a side-channel rather than in the token array so that all existing preserves_tokens proofs remain valid unchanged.

                    • definedAnchors : Array String

                      Anchor names defined in the current document (§7.1). Pushed by scanAnchorOrAlias when isAnchor = true. Checked by scanAnchorOrAlias when isAnchor = false. Reset on document boundaries (scanDocumentStart, scanDocumentEnd).

                          State Invariants #

                          The following predicate captures the structural invariants that every well-formed ScannerState must satisfy. Individual operations preserve these invariants; see Proofs/ScannerContracts.lean for proofs.

                          source
                          def L4YAML.Scanner.ScannerState.WellFormed (s : ScannerState) :
                          Prop

                          Structural well-formedness invariant for ScannerState.

                          These six conjuncts capture the essential invariants that all scanner operations must preserve:

                          1. Indent stack non-empty — the sentinel { column := -1 } is never popped.
                          2. Flow level = flow stack size — flowLevel and flowStack stay in sync.
                          3. Simple key stack = flow stack size — simpleKeyStack tracks flow nesting.
                          4. Offset bounded — the scanner never reads past the input end.
                          5. Indent stack monotonic — consecutive entries have strictly increasing columns.
                          6. Sentinel preserved — the bottom entry is always { column := -1, isSequence := false }.
