## Technical Details

### skipToContentLoop Structure

```lean
def skipToContentLoop (s : ScannerState) (fuel : Nat) : Except ScanError ScannerState :=
  match fuel with
  | 0 => .ok s
  | fuel' + 1 => do
    let mut s' := s
    -- ... whitespace/comment/newline handling ...
    match s'.peek? with
    | some c =>
      if isLineBreak c then
        s' := consumeNewline s'
        if !s'.isInFlowSequence then
          skipToContentLoop { s' with simpleKeyAllowed := true } fuel'
        else
          skipToContentLoop s' fuel'
      else .ok s'
    | none => .ok s'
termination_by fuel
```

**Key properties**:
1. Structurally recursive on `fuel` parameter
2. Base case: returns input state unchanged
3. Recursive case: processes one line, recurses with `fuel' = fuel - 1`
4. Termination proof automatic via `termination_by fuel`

**Why this enables proofs**:
- Induction on fuel provides natural proof structure
- No imperative mutation to reason about
- Each recursive call is clearly smaller (fuel decreases)

### Unexpected Challenges

#### 1. The Cascade Discovery (Most Significant)

**Initial assumption**: Refactor skipToContent → prove skipToContent_preserves_tokens → done

**Reality discovered**:
```

    requires: skipSpaces_preserves_tokens
    requires: skipWhitespace_preserves_tokens
    requires: skipToEndOfLine_preserves_tokens
    requires: consumeNewline_preserves_tokens  ✅ (proven - no loop)
```

**The problem**: skipSpaces, skipWhitespace, and skipToEndOfLine ALL use imperative for-loops!

**Code inspection**:
```lean
def skipSpaces (s : ScannerState) : ScannerState := Id.run do
  let mut s' := s
  let fuel := s.inputEnd - s.offset
  for _ in [:fuel] do        -- ❌ Imperative loop
    match s'.peek? with
    | some ' ' => s' := s'.advance
    | _ => break
  return s'
```

**Impact**: What looked like "one refactoring unblocks one proof" actually revealed **3 more refactorings needed**.

**Lesson**: Proof dependencies cascade through helper functions. Infrastructure requirements are often deeper than surface inspection reveals.

#### 2. Helper Lemma Approach Hits Wall

**Attempted approach** (ScannerCorrectness.lean:347-381):

### Unexpected Simplifications

#### 1. consumeNewline Proof Was Immediate ✅

**Expected**: Complex proof with case analysis

**Reality**: 12 lines, completely straightforward:
```lean
theorem consumeNewline_preserves_tokens (s : ScannerState) :
    (consumeNewline s).tokens = s.tokens := by
  unfold consumeNewline
  split
  · exact advance_preserves_tokens s     -- LF case
  · dsimp only []                        -- CR case
    split
    · rw [advance_preserves_tokens, advance_preserves_tokens]  -- CRLF
    · exact advance_preserves_tokens s                         -- CR alone
  · rfl                                  -- no newline
```

**Why it worked**: consumeNewline uses pattern matching (not imperative loops) and only calls advance.

**Lesson**: Functions without imperative loops are **dramatically easier** to prove. The difference is not gradual - it's binary.

#### 2. Proof Scaffolding with Sorry

**Pattern**:
```lean
-- Scaffolding: document what's needed
```

**Value**:
- Makes dependencies explicit
- Allows parallel work (different people/sessions can tackle different helpers)
