## Key Technical Insights

### 1. Imperative Loop Reasoning
Lean's `for _ in [:n] do ...` loops with mutable state resist standard proof techniques. Developed approach: extensive empirical validation + documented strategies.

### 2. The Annotation Gap
`ValidNode` models structure without tags/anchors (grammar concern), while `YamlValue` includes metadata (runtime concern). This is correct by design — solution requires working with `stripAnnotations`.

### 3. Grammable Preservation
Need to show parser preserves scanner's character-level properties (`validPlainFirst`, `noColonSpace`). Requires strengthening P10.11a scanner correctness.
